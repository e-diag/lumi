# Отчёт: доработка до каталога тарифов и операционной модели

## Что уже было до этой итерации

- Интеграция **3x-ui** (`internal/infrastructure/xui`), синхронизация клиента при подписке, поля пользователя `PanelClientUUID` / `PanelSubID`.
- Выдача подписки через публичный URL 3x-ui или fallback по нодам.
- Telegram-бот: Happ / v2RayTun, профиль, оплата по tier+дням, рефералы, антиабьюз по IP.
- ЮKassa с идемпотентной активацией (`PaymentActivation`, claim/consume).
- Веб-панель: пользователи, ноды, платежи, роутинг.

## Что сделано в этой итерации

### База данных и домен

- Модели **`Plan`**, **`ProductSettings`** (строка `id=1`), **`VPNServer`** (каталог для админки).
- Поле **`payments.plan_id`** (опционально) для связи с тарифом из БД.
- Автомиграция GORM для новых таблиц.
- **`seedCatalog`**: создаёт `product_settings`, при пустой таблице `plans` — шесть тарифов (basic/premium × 30/90/365 дней) с ценой через **`domain.SubscriptionPriceKopeks`** (та же формула, что и у «ручного» расчёта оплаты).

### Оплата

- **`PaymentUseCase.ListActivePlans`**, **`CreatePaymentByPlanCode`** — оплата по `plans.code`, метаданные ЮKassa дополняются `plan_id` / `plan_code`.
- **`CreatePayment(tier, days)`** сохранён для обратной совместимости и fallback в боте, если активных планов нет.

### Telegram-бот

- **«Купить подписку»**: при наличии планов в БД — список кнопок по тарифам; иначе — прежний выбор tier + срок.
- Callback **`u:pl:{code}`** (укладывается в лимит 64 байта для типичных кодов).
- Триал и реферальный бонус читаются из **`product_settings`** (fallback 3 / 3).
- Глобальный лимит триалов за 24 ч: **`bot.trial_global_cap_per_24h`** + **`CountTrialGrantsGloballySince`**.
- Поддержка: URL из БД приоритетнее `support_url` из YAML.
- Текст приветствия учитывает **`TrialDays`** и отдельное сообщение при **`TrialSkippedGlobal`**.

### Веб-панель

- Разделы **«Тарифы»**, **«Настройки»**, **«Серверы»** (каталог записей, не замена `XUI_*` в конфиге).
- HTMX-формы: вкл/выкл плана, смена цены в копейках, сохранение настроек продукта, CRUD-облегчённый для серверов.
- Дашборд: карточка **«записей каталога серверов»**.

### Прочее

- Расчёт цены вынесен в **`internal/domain/subscription_pricing.go`**, чтобы сид и use case не расходились.

## Что сохранено без удаления

- Вся логика 3x-ui, подписок, ЮKassa, нод и роутинга.
- Старый сценарий оплаты tier+период в боте как fallback.

## Ограничения и следующие шаги

- **Один inbound** в рантайме по-прежнему из `config` / env; таблица `vpn_servers` — учёт и UI, не автоматический multi-panel провижининг.
- Health 3x-ui в панели не опрашивается автоматически (поля `Healthy` / `LastCheckedAt` заполняются минимально при переключении «активен» в каталоге).
- Рекомендуется прогнать ручной сценарий: триал → покупка по кнопке плана → webhook → импорт в Happ / v2RayTun.

## Первый ручной тест после деплоя

1. `/start` новым пользователем — триал по `trial_days` из настроек.  
2. «Купить подписку» — выбрать план из списка — оплатить тестовой картой ЮKassa — дождаться активации.  
3. Проверить клиента в 3x-ui и ссылку `/sub/{token}`.
