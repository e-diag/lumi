# Контракт маршрутизации для мобильного клиента FreeWay

## Правила приоритета (от высшего к низшему)

1. **Direct** (наивысший приоритет)
   - Домены из `routing/lists.direct`
   - ВСЕГДА идут напрямую, без VPN
   - НЕ перенаправляются через CDN даже если основные ноды недоступны
   - Причина: избежать аномального трафика через белые IP-подсети

2. **ProxyUSA**
   - Домены из `routing/lists.proxy_usa`
   - Через USA-ноду (AI-сервисы)
   - При недоступности USA -> CDN (но НЕ direct домены)

3. **ProxyEU**
   - Домены из `routing/lists.proxy_eu`
   - Через EU-ноду (заблокированные сайты)
   - При недоступности EU -> CDN (но НЕ direct домены)

4. **CDN-fallback** (при белых списках)
   - Только для ProxyEU и ProxyUSA доменов
   - НИКОГДА не для Direct доменов

5. **Всё остальное -> Direct**
   - Домены не в списках -> напрямую

## Алгоритм выбора ноды на клиенте

```text
для каждого запроса к домену D:
  если D в direct_list:
    -> соединение напрямую (ВЫХОД)
  если D в proxy_usa_list:
    -> попробовать USA-ноду
    -> если недоступна: попробовать CDN
    -> если CDN недоступен: ОШИБКА (не fallback на direct!)
  если D в proxy_eu_list:
    -> попробовать EU-ноду
    -> если недоступна: попробовать CDN
    -> если CDN недоступен: ОШИБКА
  иначе:
    -> соединение напрямую
```

## Почему нельзя direct через CDN

Во время белых списков операторы РФ применяют CIDR-фильтрацию.
CDN-нода работает через белые IP Яндекса. Если через эти IP идет
трафик к российским сайтам (vk.com, gosuslugi.ru) — это аномалия,
которую операторы могут зафиксировать и заблокировать IP-подсеть.
Результат: CDN-нода перестает работать для всех пользователей.
