# Поддерживаемые клиенты: Happ и v2RayTun

Продукт ориентирован на **Telegram-first** сценарий: пользователь получает **ссылку подписки** (`BASE_URL/sub/{sub_token}`), которая отдаёт контент в формате, ожидаемом клиентами на базе Xray/V2Ray.

## Happ (iOS / macOS)

- **Формат**: стандартная **subscription URL** (тело ответа — обычно base64 со списком `vless://` / других URI).
- **Импорт**: в приложении — «Подписка» → добавление по URL; либо копирование ссылки из бота в буфер.
- **Кнопка в боте**: открывает HTTPS `BASE_URL/sub/...` в браузере; далее импорт из буфера в Happ.

## v2RayTun (Android)

- **Формат**: тот же **subscription URL** (совместим с v2rayNG-подобным импортом).
- **Импорт**: «Группы» → «+» → подписка по URL.
- **Диплинк в боте**: используется схема `v2raytun://install-config?url=` с base64-кодированием URL (как у ряда форков). Если на устройстве схема не зарегистрирована — используйте ручное копирование ссылки.

## Источник данных

1. **Предпочтительно**: панель **3x-ui** — после синхронизации у пользователя заполняются `PanelSubID`; backend забирает подписку с публичного sub-сервера панели (`XUI_PUBLIC_SUBSCRIPTION_BASE_URL`).
2. **Fallback**: локальная генерация VLESS из PostgreSQL (`nodes` / `node_inbounds`), если 3x-ui не настроен или у пользователя ещё нет `PanelSubID`.

## Лимит устройств

В панели для клиента выставляется **`limitIp: 3`** (одна подписка — до трёх устройств).
